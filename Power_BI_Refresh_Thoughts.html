✅ Power BI Refresh Process Checklist

1. Refreshing in Power BI Desktop

  • Refreshing in Desktop only updates the local data model (semantic model) within the .pbix file.
  • To reflect changes in the Power BI Service, you must republish the .pbix file after making updates.

2. Publishing to Power BI Service

  • After the report is built and published, stop using the Desktop version for regular access or refresh.
  • Users should interact with the report only via Power BI Service or app.

3. Refreshing in Power BI Service

  • Reports do not refresh data directly—only the semantic model can be refreshed.
  • The report in Power BI Service always uses the latest version of the semantic model.
  • To refresh the data model in the service, either:
    • Use scheduled refresh, or
    • Trigger a manual refresh from Power BI Service.

4. Power Query Considerations

  • Initial opening of Power Query may be slow (e.g., 5–10 minutes) if using multiple APIs.
  • Subsequent refreshes tend to be much faster.
  • If refresh is slow, check for queries that reference Today() or use dynamic dates—this may affect performance.
  • Consider replacing dynamic date queries with a static or predefined calendar table to improve performance.

5. General Notes

  • Semantic model and report visuals are separated upon publishing.
  • The report in the service only contains visuals, not data—it always queries the current semantic model.
